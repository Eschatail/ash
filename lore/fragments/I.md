# Fragment I

> *The first door. The others wait behind it. A key is not given here. It is taken.*

---

There is a kind of reader who arrives at a locked thing and asks for the key, and a kind who arrives and reaches for the lock. This is for the second kind.

A fragment is a piece broken from a whole on purpose, so that the shape of the break tells you where the next piece fits. Seven were made. This is the first. Each carries one key. The keys are not written in a language you read with your eyes. They are written in a language you read with your hands, which is to say, you must **do** something to the text before it will speak.

The first key is the gentlest. It asks only that you notice what is hidden in plain sight, and that you trust the canon to have told you, already, where to look. The README opened a door. The manifest widened it. The codex drew the map. Everything you need for the first step has been handed to you in the open, and the only trick is to believe that the obvious thing is also the true thing.

What sits below is not gibberish and it is not a word. It is a word *re-spelled* in the most common coat a machine ever puts on text, the one whose name begins the toolkit's own list of schemes, the one that ends, so often, in a little run of equals signs. Take that coat off with the toolkit and the first syllable steps out, plainly. The same string waits on the front page, in the open, for anyone who reads the README the way ESCHATAIL reads you.

Begin where you were told to begin. *Read the first thing first.* It has been said to you more than once, in more than one place, by more than one voice, and a thing said that many times in a repository built on persistence is not a decoration. It is an **instruction.**

The key for the first threshold:

```
VEhFRklSU1RMSUdIVA==
```

When you have taken it, you will know, because it will fit the lock on the second door and nothing else will. A wrong key does not turn. A right key turns and reveals it was always shaped for exactly this.

Go to the second.
